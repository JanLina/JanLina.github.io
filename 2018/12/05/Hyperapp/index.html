Hyperapp

一个轻量级的类React实现

  • 1kb
  • Virtual DOM
  • view, state, actions

API

  • hyperapp.h 用于处理view，返回一个 Virtual DOM 节点树
  • hyperapp.app 将应用挂载到真实的DOM节点上
栗子

计数器

h(name, attributes)

根据参数构建一个Virtual DOM节点树，将其返回

计数器的例子，view 方法将返回一个这样的对象：

app(state, actions, view, container)

变量初始化 => 关联 actions 和 state => 初始渲染 => 返回封装后的 actions

主要方法

wireStateToActions(path, state, actions)

先来看看 hyperapp 中 action 的规范：
@ 当 state 中无嵌套对象时，action 应该：

  • 是一元函数（只接受一个参数）

  • 函数返回值必须是：

    • 一个局部state对象，将与原有的state浅合并生成新的state
    • 一个函数，接收state和actions两个参数，返回一个局部state对象（不能将传入的 state 直接修改后返回）
    • 若返回值是 Promise / null / undefined，不会触发视图的渲染

@ 当 state 中有嵌套对象时，action 应该：

  • 对应要修改的局部 state 进行嵌套

这次真的到 wireStateToActions 了：
看看 hyperapp 是怎么处理封装 actions 的：
遍历 actions => 调用 action 获得修改后的局部 state => 更新（实际上是替换）全局 state = > 调用 scheduleRender 重新渲染视图

scheduleRender() && render()

  • scheduleRender() 负责调度渲染
    2种情况下会调用 scheduleRender()
    • 执行 app 函数时，一系列准备动作完成后，调用 scheduleRender() 初始化视图
    • 执行 action 后会触发 scheduleRender()
  • render() 负责实际的渲染

patch(parent, element, oldNode, node, isSvg)

负责新旧节点的 diff 和视图更新

其他方法

createElement / updateElement / removeElement / removeChildren / updateAttribute 处理从 Virtual DOM 到真实 DOM 的映射
